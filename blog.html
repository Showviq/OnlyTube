Discuss the scope of var, let, and const!

VAR :

  • Function Scope : Variables declared with var are confined to the function where they are defined, limiting their accessibility.
  • Hoisting : var-declared variables are hoisted to the top of their containing function or global scope. This allows pre-declaration use, but they are initialized with undefined until formally declared.

LET :

  • Block Scope : Variables declared with let have block-level scope, ensuring accessibility only within the specific block (e.g., a function, loop, or code enclosed in curly braces {}) where they are defined.
  • Hoisting : Hoisting for let-declared variables brings them to the top of their block, without initialization. Using them before declaration is not allowed.

CONST :

  • Block Scope : Similar to let, const is block-scoped.
  • Immutability : const variables are designed as constants; their values cannot be reassigned after initialization. However, for objects and arrays declared with const, modifications to properties or elements are allowed; only reassigning the variable itself is prohibited.
Tell us the use cases of null and undefined!

null :

  • Explicit Absence of Value : null serves as a representation of the deliberate absence of any object value or value in general. It is commonly employed to signify that a variable or property should lack a value or has been intentionally cleared.
  • As a Placeholder : Additionally, null can function as a placeholder or initial value that may later receive an assigned value in your code.
  • Returned by Functions : Certain functions or methods might utilize null as a return value to signify that they did not find a meaningful result.

undefined :

  • Default Value : In JavaScript, variables that are declared but not assigned a value are automatically initialized with undefined. Therefore, undefined is commonly employed as the default value for function parameters and variables.
  • Property Access : When attempting to access an object property that does not exist, the result will be undefined.
  • Array Elements : Accessing an array element beyond the current length of the array will yield undefined.
  • Function Return : If a function does not explicitly provide a return value, it automatically returns undefined.
What do you mean by REST API?

REST API :

A REST API is an interface for software applications that aligns with the principles of the REST architectural style, facilitating communication with RESTful web services. The internet comprises interconnected networks, and within this framework, a web service denotes a collection of open protocols and standards designed to facilitate data exchange between client-server applications. Specifically, web services that adhere to the principles of REST architecture are referred to as RESTful web services.
